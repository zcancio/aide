## Files Created

### 1. `evals/scripts/eval_compare.py` (132 lines)
Main CLI tool with the following functionality:
- Loads `report.json` from two eval run directories
- Matches scenarios by name between runs
- Calculates score deltas and token usage changes
- Prints formatted comparison table with:
  - Individual scenario scores (v1 vs v2)
  - Delta percentages
  - Token usage changes with directional arrows (↓/↑/→)
  - Overall summary row
- Highlights regressions (>2% score drop)
- Highlights improvements (>2% score gain)

**Key Functions:**
- `compare_runs(run_a, run_b)` - Main comparison logic
- `match_scenarios(a_results, b_results)` - Match scenarios by name
- `print_comparison_table(scenarios, run_a_name, run_b_name)` - Pretty print table
- `print_overall_row(...)` - Print overall summary
- `main()` - CLI entry point with argparse

## Design Decisions

1. **Report Format Compatibility**: Tool reads the existing `report.json` structure generated by `eval_multiturn.py` with no modifications needed to existing code.

2. **Box Drawing Characters**: Used Unicode box drawing characters for professional table output (╔═╗║╠╣╚╝╪╧).

3. **Directional Arrows**: Token usage changes show arrows (↓ for decrease, ↑ for increase, → for no change) for quick visual scanning.

4. **2% Threshold**: Regressions and improvements use >2% threshold to filter out noise and focus on meaningful changes.

5. **Scenario Matching**: Matches scenarios by name, gracefully handles mismatches (only compares scenarios present in both runs).

6. **Pure Functions**: Core logic (match_scenarios, print functions) kept as pure functions for testability.
